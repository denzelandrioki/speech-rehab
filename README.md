# Speech Rehab (Речевая реабилитация)

Android-приложение для тренировки узнавания слов по картинкам: крупный интерфейс, минимум шагов, офлайн-кэш изображений, учёт статистики и два режима тренировки. Проект в экосистеме [TechLabHub](https://techlabhub.ru). **Application ID:** `ru.techlabhub.speechrehab`.

| Параметр | Значение |
|----------|----------|
| **Версия** | `0.1.0-beta1` (`versionCode` 2) |
| **minSdk** | 26 |
| **targetSdk / compileSdk** | 35 |

## Возможности

### Главный экран

- Запуск **ассистированной тренировки** (картинка + «верно» / «неверно» / «следующая»).
- Запуск **самостоятельного режима** — выбор **одного из четырёх** слов по картинке; после ответа вместо изображения показывается **правильное слово** (зелёный текст при верном выборе, красный при ошибке). В альбомной ориентации варианты в сетке **2×2**, картинка не перекрывает кнопки.
- Переходы к **статистике**, **настройкам**, **словарю**.

### Ассистированная тренировка

- Карточка «изображение + слово»; опционально **письменная подсказка** слова (из настроек).
- Отметки **верно** / **неверно**, затем **следующая** карточка.

### Настройки

- **Язык интерфейса** (системный / русский / английский) — per-app locale через AppCompat + `locales_config.xml`.
- **Язык текста на карточке** (русский / английский / оба / без подписи с fallback).
- Режим подбора карточек (случайный, смешанный, только новые и т.д.), размер пакета, фильтр **категорий**.
- Источники иллюстраций: **ARASAAC**, **Pixabay**, **Pexels** (внешние — при ключах в сборке).
- Политика онлайн-картинок, приоритет локальных/удалённых данных, **режим ротации изображений** (варианты на слово в Room).

### Словарь и статистика

- **Словарь** — слова по категориям, участие в тренировке.
- **Статистика** — точность, динамика, сложные/лёгкие слова, категории; отдельно сводка по **режиму «4 варианта»** (где реализовано в репозитории статистики).

### Данные

- **Room** — категории, слова, варианты картинок, сессии, попытки assisted и multiple choice.
- **DataStore Preferences** — пользовательские настройки.
- Локальный кэш файлов изображений; при необходимости запросы к API (Pixabay / Pexels / др. по коду).

## Требования к окружению

- **Android Studio** с поддержкой AGP из проекта (см. `gradle/libs.versions.toml`, сейчас **8.7.2**).
- **JDK 17** (`compileOptions` / `kotlinOptions` модуля `app`).
- **Android SDK** с **API 35** для сборки.

## Клонирование и запуск

1. Откройте **корень** репозитория в Android Studio (один модуль `:app`, имя проекта в Gradle — **SpeechRehab**).

2. Сборка из корня:

   ```bash
   # Windows
   .\gradlew.bat :app:assembleDebug

   # Linux / macOS
   chmod +x gradlew
   ./gradlew :app:assembleDebug
   ```

3. **`local.properties`** — обычно создаётся IDE; укажите `sdk.dir=...`. Файл в `.gitignore`.

## Ключи API (Pixabay / Pexels)

В **`local.properties`** (в корне проекта):

```properties
pixabay.api.key=ВАШ_КЛЮЧ_PIXABAY
pexels.api.key=ВАШ_КЛЮЧ_PEXELS
```

Ключи попадают в **`BuildConfig`** при сборке. **Без ключей** приложение собирается и работает: остаются **ARASAAC** и уже закэшированные изображения; Pixabay/Pexels не используются до появления ключей.

После правок выполните **Sync Project with Gradle Files**.

## Архитектура (кратко)

- **`app/.../ui`** — Compose-экраны, ViewModel.
- **`app/.../domain`** — модели, use case-ы (следующая карточка, следующий вопрос MC, статистика и т.д.).
- **`app/.../data`** — Room, сеть, файловый кэш, реализации репозиториев, DataStore.

Детали по классам — в **KDoc** в исходниках.

## Данные и конфиденциальность

- База и кэш — в **приватном каталоге** приложения на устройстве.
- Ключи API не коммитьте: только **`local.properties`** или секреты CI.

## CI

В репозитории: **`.github/workflows/ci.yml`** — сборка и тесты на push/PR (ветки настраиваются в workflow).

## Известные ограничения при разработке

- На Windows иногда падает шаг **`processDebugResources`** с ошибкой **«Couldn't delete … R.jar»** — файл занят другим процессом (IDE, антивирус). Закройте лишние Gradle/Studio или перезапустите сборку.
